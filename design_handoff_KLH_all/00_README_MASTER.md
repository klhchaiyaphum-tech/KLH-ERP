# KLH — แพ็กเกจ Handoff รวมทุกระบบ (สำหรับทีมพัฒนา / Claude Code)

> เอกสารแม่ของชุดดีไซน์ KLH ทั้งหมด — back-office, POS หน้าร้าน และ LINE OA
> เป้าหมาย: นำดีไซน์ไปสร้าง/ปรับใน **โค้ดเบสจริง โดยไม่ทำให้ฟังก์ชันเดิมพัง**

---

## ⚠️ อ่านก่อนเริ่ม — กฎทองของการ handoff

ไฟล์ในแพ็กเกจนี้คือ **ดีไซน์อ้างอิง (HTML + React/Babel ในเบราว์เซอร์)** ที่ใช้แสดง "หน้าตา + พฤติกรรม UI ที่ต้องการ" — **ไม่ใช่โค้ดโปรดักชันที่จะ copy ไปแปะทับของเดิม**

**แยก 2 ชั้นนี้ให้ขาด:**
| ชั้น | คืออะไร | ทำอะไรกับมัน |
|---|---|---|
| **View (หน้าตา)** | สิ่งที่อยู่ในไฟล์ดีไซน์นี้ — layout, สี, ฟอนต์, state ของ UI, การกดแล้วเปลี่ยนจอ | **เอาไปทำใหม่ในเฟรมเวิร์กจริง** ให้หน้าตาตรงตามนี้ |
| **Logic (การทำงาน)** | API, ฐานข้อมูล, การคิดเงิน, ตัดสต็อก, สิทธิ์ผู้ใช้ ที่โค้ดเดิม **มีอยู่แล้วและทำงานได้** | **ห้ามรื้อ ห้ามเขียนใหม่** — เอา UI ใหม่ไปครอบ แล้วต่อกลับเข้าฟังก์ชันเดิม |

**ขั้นตอนที่ทำให้ของเดิมไม่พัง:**
1. **ทำทีละจอ** ไม่เปลี่ยนรวดเดียวทั้งระบบ — เสร็จจอไหนเทสจอนั้น
2. **แตก branch แยก** ค่อย merge ทีละส่วน พังแล้วย้อนได้
3. ก่อนลงมือแต่ละจอ **ลิสต์ endpoint/ฟังก์ชันเดิม** ที่จอนั้นต้องเรียก แล้ว reuse — อย่าเขียนซ้ำ
4. **ห้ามเปลี่ยนโครงสร้างข้อมูล/API เดิม** เว้นแต่จำเป็นจริง และตกลงกันก่อน
5. ทุกค่าตัวเลข/ข้อความในดีไซน์เป็น **ตัวอย่าง (mock)** — ของจริงต้องดึงจาก backend

> ใช้ไฟล์ดีไซน์เป็น "ภาพอ้างอิงเชิงโต้ตอบ" (เปิดในเบราว์เซอร์กดเล่นได้) + README แต่ละระบบเป็น "สเปก"

## จุดต่อระบบจริง (Integration Points) — สรุปต่อระบบ

### 1) Back-office (macOS)
- ทุกโมดูลดึง/บันทึกผ่าน API เดิม: catalog, สต็อก, สมาชิก, เจ้าหนี้, ใบกำกับ
- **WMS** รับเข้า/โอน → อัปเดตสต็อก + แจ้งเตือน LINE กลุ่มจัดซื้อ (ROP)
- **Invoice OCR** → ต่อ OCR engine จริง แล้วส่งเข้า WMS + บัญชี
- **บัญชี AP/AR/VAT** — เมนูย่อยยัง **TODO** (วางโครง overview ไว้) ผูกกับ Supplier (AP), Customer/AR (AR), OCR (VAT)

### 2) POS Handheld
- เลือกสินค้า → สร้างออเดอร์ → **พิมพ์บิล QR สถานะ "ยังไม่ชำระ"** → ลูกค้าถือไปจ่ายที่แคชเชียร์
- พักบิล/เปิดบิลใหม่/เรียกบิลเดิม = snapshot ออเดอร์ใน order service
- สมาชิก → member lookup (ส่วนลดตามระดับ)

### 3) POS PC / T2 Lite / แคชเชียร์ — ★ Tigercashbox
- `tigercashbox.openDrawer()` — เปิดลิ้นชัก (ESC/POS kick หรือ SDK)
- `tigercashbox.print(payload)` — พิมพ์ใบเสร็จ/บิล
- `payment.createQR(amount) → {qrPayload, ref}` + `payment.onConfirmed(ref, cb)` — QR PromptPay จริง + callback ยืนยัน
- `devices.status() → {drawer, printer, scanner}` — ผูกไฟสถานะอุปกรณ์
- จอลูกค้า T2 = จอที่ 2 แสดงรายการ/QR เรียลไทม์

### 4) LINE OA (LIFF)
- **Rich Menu** 6 ช่อง → เปิด LIFF app (`liff.init` / `liff.login`)
- **Login เบอร์โทร + OTP** → LINE Login + ผูกเบอร์เข้ากับ member เดิม
- **QR PromptPay** → payload **EMVCo/PromptPay จริง** จาก payment gateway (ไม่ใช่ QR ลายจำลองในดีไซน์)
- **ตรวจสลิป** → **Slip Verify API** (จับคู่กับ statement/เลขอ้างอิง) → auto-confirm; ถ้าไม่ตรง = คิวให้แอดมินตรวจมือ
- **ชำระสำเร็จ → พิมพ์ Pick List ที่หลังร้านอัตโนมัติ + ตัดสต็อก WMS** (จุดเชื่อมกับระบบหลังบ้าน)
- **ใบรายการ** = ลูกค้ากดพิมพ์/บันทึกเองได้ (80mm หรือ A4/PDF) — `window.print()` ในดีไซน์คือ placeholder ของการ render บิลจริง
