# Handoff: POS หน้าร้าน — PC + Sunmi T2 Lite + แคชเชียร์ (Tigercashbox)

> เอกสารส่งงานให้ทีมพัฒนา (Claude Code) นำดีไซน์ไปสร้างในระบบ POS จริง

---

## Overview

ชุดหน้าจอ **เครื่องคิดเงินหน้าร้าน (counter / cashier POS)** สำหรับ **ร้านค้าปลีก/ส่ง** ภาษาไทยล้วน 2 รูปแบบเครื่อง + หน้าชำระเงินแคชเชียร์ที่ใช้ร่วมกัน:

1. **POS สำหรับ PC** — เดสก์ท็อปจอกว้างแนวนอน คุมด้วยเมาส์ เลย์เอาต์ 3 คอลัมน์
2. **Sunmi T2 Lite** — เครื่องตั้งเคาน์เตอร์ 2 จอ (จอขายแบบสัมผัส + จอลูกค้า/Customer Display)
3. **หน้าชำระเงินแคชเชียร์** — เงินสด (คีย์แพด + ลิ้นชัก) / QR พร้อมเพย์ / บัตร — **เตรียมเชื่อม Tigercashbox** (ลิ้นชักเก็บเงิน + เครื่องพิมพ์ + สแกนเนอร์)

ทั้งสองเครื่องรองรับ **พักบิล / เปิดบิลใหม่ / เปิดบิลเดิม** (hold & resume หลายบิลพร้อมกัน) และ **สมาชิก** (ส่วนลดตามระดับ + แต้ม) — ใช้ design system + แคตตาล็อกสินค้าชุดเดียวกับชุด Handheld

> ชุดนี้คือ "ปลายทาง" ของโฟลว์: Handheld รับออเดอร์ → พิมพ์บิล QR → ลูกค้านำมาชำระที่เครื่องแคชเชียร์ (PC / T2 Lite) ชุดนี้

## About the Design Files

ไฟล์ในแพ็กเกจนี้คือ **ดีไซน์อ้างอิงที่สร้างด้วย HTML + React (Babel ในเบราว์เซอร์)** — เป็น *prototype* แสดงหน้าตา/พฤติกรรมที่ต้องการ **ไม่ใช่โค้ดโปรดักชันที่จะนำไปใช้ตรงๆ**

งานของผู้พัฒนา: **สร้างขึ้นใหม่ในโค้ดเบส/เฟรมเวิร์กของระบบ POS จริง** (React/Electron, .NET/WPF, Flutter, Android ฯลฯ) ด้วย component และ pattern ที่โปรเจกต์มีอยู่ ใช้เอกสารนี้เป็น "spec" และไฟล์ HTML เป็น "ภาพอ้างอิงเชิงโต้ตอบ" (เปิด `POS PC & Sunmi T2 Lite.html` ในเบราว์เซอร์เพื่อกดเล่น)

**The HTML is a reference, not shippable code.**

## Screens / Views

### A) POS สำหรับ PC — `<PCPos>` (desktop.jsx)

หน้าต่างแอปเดสก์ท็อป กว้าง **1180px**, สูง **752px** (เลย์เอาต์คงสัดส่วน ปรับ responsive ได้):

1. **OS title strip** (h30) — แถบแอป (traffic-light dots) + ชื่อระบบ — ในแอปจริงคือ title bar ของ window
2. **Top bar** (h56, `.topbar`) — โลโก้ร้าน (`.brandmark`) + ชื่อ/สาขา · เส้นคั่น · **แท็บบิล (`OrderTabs`)**: แท็บบิลปัจจุบัน (จุดสีแบรนด์) + แท็บบิลที่พักไว้ (คลิกเพื่อ resume, โชว์ `#เลขบิล · ยอด`) + ปุ่ม **＋ บิลใหม่** · ขวาสุด: chip **Tigercashbox** (สถานะเชื่อมต่อ) + ชื่อแคชเชียร์ + เวลา/กะ
3. **Body 3 คอลัมน์:**
   - **ซ้าย (176px):** รายการหมวดหมู่แนวตั้ง (`.dcat`) — มีจุดสีประจำหมวด, active = พื้นขาว + ตัวอักษรสีแบรนด์ + แถบซ้าย
   - **กลาง (ยืด):** ช่องค้นหา/สแกนบาร์โค้ด (โชว์จำนวนรายการ) + **ตารางสินค้า** `ProductGrid` 3 คอลัมน์ — การ์ด `.pcard` คลิกเพื่อ +1, มุมขวาบนโชว์ badge จำนวนเมื่ออยู่ในบิล, hover ขอบสีแบรนด์
   - **ขวา (376px):** `OrderPanel` — หัวบิล `#เลขบิล` + จำนวนชิ้น, แถวสมาชิก (`MemberRow`), รายการสินค้าในบิล (ชื่อ · ราคา×จำนวน · `Stepper` +/− · ยอดรวมแถว · ปุ่ม ✕ ลบ), ท้าย: ยอดรวม / ส่วนลดสมาชิก / **ยอดสุทธิ** (ตัวใหญ่สีแบรนด์) + ปุ่ม **พักบิล** (ghost) และ **ชำระเงิน** (primary)
4. **Payment overlay** (`PaymentOverlay`) — เปิดทับเมื่อกดชำระเงิน (ดูหัวข้อ C)

### B) Sunmi T2 Lite — `<SunmiT2>` (desktop.jsx)

เครื่องตั้งเคาน์เตอร์ **2 จอ**:

- **จอขายหลัก** (`.term-screen` 1000×620 ในกรอบเครื่อง + ขาตั้ง) — โครงคล้าย PC แต่ **จอสัมผัส ปุ่มใหญ่ขึ้น** และจัด 2 คอลัมน์ (ไม่มี rail ซ้าย ใช้ **ชิปหมวดแนวนอน** แทนเพื่อประหยัดพื้นที่):
  - Top bar: ร้าน/แคชเชียร์ + ปุ่ม **＋ บิลใหม่** + ปุ่ม **บิลพัก (badge จำนวน)** (คลิก resume บิลล่าสุด) + chip Tigercashbox
  - ซ้าย: ค้นหา + ชิปหมวด + `ProductGrid` (`big` = การ์ดใหญ่ขึ้น, แตะง่าย)
  - ขวา (348px): `OrderPanel` เหมือน PC
- **จอลูกค้า (Customer Display)** (`.cust-screen` 300×470, ธีมเข้ม) — แยกอีกจอหันหาลูกค้า:
  - ปกติ: โลโก้/ชื่อร้าน + **รายการสินค้าเรียลไทม์** + ยอดสุทธิตัวใหญ่
  - ตอนกดชำระเงิน: แสดง **QR + ยอด** ให้ลูกค้าสแกนจ่ายเอง

### C) หน้าชำระเงินแคชเชียร์ — `<PaymentOverlay>` (desktop.jsx) ★ จุดเชื่อม Tigercashbox

โมดอลกลางจอ (`.ovl` + `.pay`) 2 ฝั่ง:

- **ฝั่งซ้าย (พื้นเข้ม 300px):**
  - **ยอดที่ต้องชำระ** (ตัวเลขใหญ่) + เลขบิล/จำนวนชิ้น/สมาชิก
  - **chip Tigercashbox · เชื่อมต่อแล้ว** (ไฟเขียว)
  - รายการอุปกรณ์ (`DevRow`) พร้อมสถานะไฟ: **ลิ้นชักเก็บเงิน** (เปิด/ปิด) · **เครื่องพิมพ์ใบเสร็จ** (พร้อม) · **เครื่องสแกนบาร์โค้ด** (พร้อม)
  - ปุ่ม **เปิดลิ้นชัก** (สั่งเปิดลิ้นชักด้วยมือ)
- **ฝั่งขวา:** หัวข้อ "รับชำระเงิน" + ปุ่มปิด, แท็บวิธีชำระ (`.ptab`): **พร้อมเพย์ QR** / **เงินสด** / **บัตร**
  - **QR:** แสดง QR (decorative) + ข้อความให้ลูกค้าสแกน + chip "รอ Tigercashbox ยืนยันการชำระ…" + ปุ่ม **ยืนยันรับชำระ**
  - **เงินสด:** กล่อง **รับเงิน / เงินทอน** (คำนวณอัตโนมัติ) + ปุ่มเงินด่วน (พอดี/100/500/1000) + **คีย์แพดตัวเลข** (1–9, 00, 0, ⌫) + ปุ่ม **ยืนยันรับเงิน · เปิดลิ้นชัก** (กดได้เมื่อรับเงิน ≥ ยอด → เปิดลิ้นชัก)
  - **บัตร:** prompt "สอด/แตะบัตรที่เครื่อง EDC" + ยืนยัน
  - **หน้าสำเร็จ:** เครื่องหมายถูกเขียว + ยอดรับชำระ + เงินทอน (ถ้าเงินสด) + chip "ลิ้นชักเปิดแล้วผ่าน Tigercashbox" + ปุ่ม **พิมพ์ใบเสร็จ** / **เสร็จสิ้น** (เคลียร์บิล)

## Interactions & Behavior

| Action | Result |
|---|---|
| คลิกการ์ดสินค้า | +1 ลงบิลปัจจุบัน, badge จำนวนอัปเดต |
| Stepper / ✕ ในบิล | ปรับ/ลบจำนวน |
| เลือกหมวด / พิมพ์ค้นหา | กรองสินค้า (ชื่อ หรือ บาร์โค้ด) |
| สมาชิก (`MemberRow`) | ผูก/ปลดสมาชิก → ส่วนลดตามระดับคำนวณอัตโนมัติ (prototype สลับ DEMO_MEMBER; ของจริงต่อ member lookup) |
| **พักบิล** | snapshot บิลปัจจุบัน (รายการ+สมาชิก+เวลา) เข้า `parked[]`, เคลียร์บิล, เลขบิลถัดไป |
| **＋ บิลใหม่** | ถ้าบิลปัจจุบันมีของ → พักก่อน แล้วเปิดบิลเปล่า |
| คลิกแท็บบิลพัก (PC) / ปุ่มบิลพัก (T2) | resume บิลนั้น (ถ้าบิลปัจจุบันมีของจะถูกพักอัตโนมัติก่อน) |
| **ชำระเงิน** | เปิด `PaymentOverlay` (และจอลูกค้า T2 แสดง QR) |
| เลือกวิธี / ใส่จำนวนเงิน | คำนวณเงินทอน; ปุ่มยืนยันเปิดเมื่อ รับเงิน ≥ ยอด (เงินสด) |
| **เปิดลิ้นชัก / ยืนยันเงินสด** | ตั้งสถานะลิ้นชัก = เปิด (จุดเชื่อม Tigercashbox) |
| ยืนยันรับชำระ → เสร็จสิ้น | หน้าสำเร็จ → เคลียร์บิล |

- ทรานสิชัน: overlay `popin`/`fadein`; การ์ด hover/active; ปุ่ม active ขยับเล็กน้อย
- ยังไม่มีใน prototype (ต้องเพิ่มตอนทำจริง): สต็อก/หมดสต็อก, ยกเลิก/คืนสินค้า, เปิด-ปิดกะ & สรุปยอด, error/timeout การชำระ, สิทธิ์พนักงาน, พิมพ์จริง

## ★ Tigercashbox — จุดที่ต้องเชื่อมต่อจริง

ใน prototype ทั้งหมดเป็น **mockup/สถานะจำลอง** — ตำแหน่งที่ต้องต่อ SDK/อุปกรณ์จริง (ดูใน `PaymentOverlay`, `DevRow`):

1. **เปิดลิ้นชักเก็บเงิน (cash drawer)** — ปุ่ม "เปิดลิ้นชัก" และตอนยืนยันเงินสด → เรียกคำสั่งเปิดลิ้นชักของ Tigercashbox (มักผ่าน printer ESC/POS kick หรือ SDK). อัปเดตสถานะ `drawer` จาก event จริง (เปิด/ปิด)
2. **เครื่องพิมพ์ใบเสร็จ** — ปุ่ม "พิมพ์ใบเสร็จ" → ส่ง job พิมพ์ (ใบเสร็จ/บิล QR) ผ่าน Tigercashbox/printer
3. **QR พร้อมเพย์** — แทน QR decorative ด้วย **payload EMVCo/PromptPay จริง** จาก payment gateway; chip "รอ Tigercashbox ยืนยันการชำระ…" ต้องผูกกับ **callback/webhook ยืนยันการชำระ** แล้วเปลี่ยนเป็นหน้าสำเร็จอัตโนมัติ
4. **สถานะอุปกรณ์ (`DevRow`)** — ผูกไฟเขียว/แดงกับสถานะการเชื่อมต่อจริงของลิ้นชัก/เครื่องพิมพ์/สแกนเนอร์
5. **chip "Tigercashbox · เชื่อมต่อแล้ว"** — สะท้อนสถานะ session/health ของอุปกรณ์จริง

> สรุป interface ที่ควรมี: `tigercashbox.openDrawer()`, `tigercashbox.print(payload)`, `payment.createQR(amount) → {qrPayload, ref}`, `payment.onConfirmed(ref, cb)`, `devices.status() → {drawer, printer, scanner}`

## State Management

- `useRegister()` (desktop.jsx) — ต่อยอดจาก `useCart` (shared.jsx): `cart` (`{id:qty}`), `member`, `parked[]` (snapshot บิล), `billNo`, `disc`/`net`, และ `park/resume/newBill`
- `PaymentOverlay` ภายใน: `method` (qr/cash/card), `recv` (เงินรับ), `drawer` (เปิด/ปิด), `paid` (สำเร็จ)
- แต่ละเครื่อง (`PCPos`/`SunmiT2`) ถือ `cat`, `q`, `pay` (เปิด overlay) แยกของตัวเอง
- **ของจริง:** ดึง catalog/ราคา/สต็อก/สมาชิกจาก backend, ผูกบิล/พักบิล/การชำระเข้ากับ order & payment service และอุปกรณ์ Tigercashbox

## Design Tokens

ใช้ token ชุดเดียวกับ Handheld (`pos.css :root`) + คลาส desktop ใน `desktop.css`:

- **Colors:** แบรนด์ส้มแดง `--brand #E94E27` / `--brand-2 #C73A1B` / `--brand-soft #FCEDE6`; พื้น `--bg #F3F1EC` / `--surface #FFFFFF`; ink `--ink #221E1A` / `--ink-2 #6E665D` / `--ink-3 #A39A8E`; เส้น `--line #ECE8E0`; สำเร็จ `--ok #3E8E4F`; เตือน `--warn #D98A0B`; แผงแคชเชียร์ฝั่งซ้ายพื้น `--ink`, ไฟอุปกรณ์เขียว `#7FE0A8`
- **Radius:** การ์ด/ปุ่ม 11–13px, โมดอล 20px, pill/badge 999px
- **Type:** **IBM Plex Sans Thai** (ข้อความ) + **IBM Plex Sans** (`--ff-num`, ตัวเลข, tabular-nums via `.num`). สเกล: 12–16 เนื้อหา, 23–31 หัวข้อ, ยอดสุทธิ 26–42
- **Currency:** `฿` นำหน้า + คั่นหลักพัน (`baht()` ใน shared.jsx)
- **Frames:** PC = `.win` (1180px) + os strip; T2 = `.term-body/.term-screen` (1000×620) + ขาตั้ง + `.cust-screen` (300×470)

## Assets

- **รูปสินค้า:** ชุด PC/T2 เป็น **การ์ดข้อความ (ชื่อ+ราคา) ไม่มีรูป** ตามสไตล์ POS หน้าร้าน SKU เยอะ — ถ้าต้องการรูปให้ดึงจากระบบเดิม
- **QR:** เป็นลายจำลอง (deterministic SVG ใน `QR` component) — **แทนด้วย QR ชำระเงินจริง**
- **ไอคอน:** inline SVG (`Ic` ใน shared.jsx + ไอคอน card/drawer/bag/× ใน desktop.jsx) — แทนด้วยไอคอนเซ็ตเดิมได้
- **ฟอนต์:** Google Fonts (IBM Plex Sans Thai/Sans) — ของจริงให้ bundle เอง · ไม่มี emoji/โลโก้ลิขสิทธิ์
