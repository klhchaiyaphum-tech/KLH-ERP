# KLH Data Survey (มือถือ) — Handoff สำหรับ Claude Code / ทีมพัฒนา

> เอาดีไซน์นี้ไปทำหน้า `page=survey` ในระบบจริง (Google Apps Script)
> **กฎเหล็ก: ห้ามแก้ลำดับขั้นตอน · ห้ามตัดปุ่ม · ห้ามแตะ logic เดิม — เอา UI ใหม่ไปครอบฟังก์ชันเดิมเท่านั้น**

---

## นี่คืออะไร
ดีไซน์ **หน้าตาใหม่ (UI/UX)** ของหน้า Data Survey เวอร์ชันมือถือ เป็น HTML + React (Babel ในเบราว์เซอร์) ใช้ **ดูเป็นภาพอ้างอิงเชิงโต้ตอบ** — เปิด `KLH Data Survey.html` ในเบราว์เซอร์กดเล่นได้ — **ไม่ใช่โค้ดโปรดักชันที่จะ copy ไปแปะทับ**

งานของ dev: ทำหน้าตา/สไตล์ตามนี้ในไฟล์ template เดิมของ GAS โดยคงโครงสร้างฟอร์ม ชื่อฟิลด์ id และการเรียกฟังก์ชันเดิมทั้งหมด

---

## ลำดับขั้นตอน (ห้ามสลับ — ตรงกับระบบจริง)
หน้าเดียวเลื่อนยาว 4 ส่วนตามลำดับ:

1. **ค้นหาสินค้า** — ช่องค้นหา (ชื่อ/บาร์โค้ดชิ้น/บาร์โค้ดลัง) + ปุ่มกล้องสแกน + ปุ่มค้นหา + ปุ่ม **เพิ่มสินค้าใหม่**
2. **ข้อมูลพื้นฐาน** — รหัสกลุ่ม(AE), นิติบุคคล(AD)+⚙️, ชื่อสินค้า(B), ขนาด(D), หมวด(C)+⚙️, ผู้จำหน่าย(G)+คัดลอก+แก้ไข, หน่วยบรรจุ(F)+⚙️, ตัวคูณ(E)
3. **🏷️ ต้นทุน & ราคาขาย** — ต้นทุนสุทธิ/ชิ้น(R) + ปุ่มคำนวณ (ป๊อปอัปเต็มสูตร) · ขายส่งเก่า(U) · ส่ง×หน่วยใหญ่(V) · ขายปลีกเก่า(W) · ปลีกคำนวณ(X) ปรับด้วยปุ่ม ▲▼ ทีละ 0.25 · เช็กบ็อกซ์ **สร้างหน่วยโหล ×12** (บาร์โค้ดโหล AH / ราคาโหล AI)
4. **รูปภาพ & บาร์โค้ด** — ถ่ายรูป(+ตัดพื้นหลัง/จัดกึ่งกลาง/เปลี่ยนสีพื้น) · สแกนบาร์โค้ด · บาร์โค้ดชิ้น(A)* · บาร์โค้ดลัง(AB) · วันที่อัปเดต(Y) · ปุ่ม **บันทึกเข้าระบบ** · ปุ่ม **ตรวจสอบ & พิมพ์ป้าย**

\* บันทึกจะกดได้ต่อเมื่อมีบาร์โค้ดชิ้น (A) แล้วเท่านั้น

## ปุ่ม/พฤติกรรมที่ต้องคงไว้ (อย่าตัด)
| UI ในดีไซน์ | ผูกกับฟังก์ชันเดิม |
|---|---|
| ช่องค้นหา + ปุ่มค้นหา | ค้นหาสินค้าจาก KLH DATA (เดิม) |
| ปุ่มกล้อง (ค้นหา / section 3) | สแกนบาร์โค้ด → ได้เลขมาเติม/ค้นหา (เดิม) |
| ปุ่ม **เพิ่มสินค้าใหม่** | สร้างแถวใหม่ + gen รหัส ITEM***** |
| ⚙️ ข้างแต่ละ dropdown | `manageListData` (จัดการนิติบุคคล/หมวด/หน่วย) |
| คัดลอก / แก้ไข ผู้จำหน่าย | ฟังก์ชันจัดการผู้จำหน่าย (เดิม) |
| ปุ่ม **คำนวณ** ต้นทุน | สูตรคำนวณ R จาก ทุนดิบ/ลดเงินสด/ลด%/แถม/ภาษี/ขนส่ง (เดิม) |
| ▲▼ ราคา | snap 0.25 แล้วเขียนคอลัมน์ U/V/W/X |
| **สร้างหน่วยโหล** | gen บาร์โค้ดโหล + ราคาโหล |
| **บันทึกเข้าระบบ** | `processAndSaveAll` (เขียนกลับ KLH DATA) |
| **ตรวจสอบ & พิมพ์ป้าย** | ฟังก์ชันพิมพ์ป้ายราคา (เดิม) |

> **สิ่งที่ต้องคงเป๊ะ:** ชื่อ/ลำดับคอลัมน์ A–AI, ชื่อฟิลด์/id ของฟอร์ม, การเรียก `google.script.run.xxx` ทั้งหมด — เปลี่ยนแค่ HTML/CSS รอบนอก

## ไฟล์ในแพ็กเกจ
| ไฟล์ | เนื้อหา |
|---|---|
| `KLH Data Survey.html` | หน้าโชว์ (เปิดในเบราว์เซอร์กดเล่นได้) |
