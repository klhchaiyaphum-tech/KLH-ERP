# ส่วนที่ 2: ฟังก์ชันเฉพาะ KLH (ที่ต้องการจริง)
# ════════════════════════════════════

## โมเดลสำคัญ: KLH = นิติบุคคลผู้เสียภาษีเพียงรายเดียว
```
ห้างหุ้นส่วนจำกัด เคแอลเอช (KLH)
  → เป็นร้านที่ "นำส่งภาษี" (จด VAT)
  → ต้องคุม: ภาษีซื้อ / ภาษีขาย / ยอดขาย / สินค้าคงคลัง

ชื่อนิติบุคคลอีก 4 ชื่อ
  → ไม่ต้องดูภาษีซื้อ ภาษีขาย ยอดเงินเข้า
  → ดูแค่ ยอดซื้อ / เจ้าหนี้ (ในชีท SHOPS)
```

## เมนูใหม่: "ประมาณการ สรรพากร" (ภพ.30)
| หน้า/ไฟล์ | URL | ทำอะไร |
|-----------|-----|--------|
| `tax_estimate.html` | `?page=tax_estimate` | สรุป ภพ.30 ประจำเดือน: VAT ซื้อ / VAT ขาย / ภาษีต้องชำระ |
| `tax_exec_input.html` | `?page=tax_exec` | หน้าผู้บริหารกรอกประมาณการ % ยอดขาย |

## 2.1 ภาษีซื้อ (Input VAT)
```
แหล่ง: KLH Invoice OCR (มีอยู่แล้ว)
  ↓
เงื่อนไข: บิลที่ผู้ซื้อ = "ห้างหุ้นส่วนจำกัด เคแอลเอช"
          AND เป็นใบกำกับภาษีเต็มรูป (มีเลขผู้เสียภาษี + VAT แยก)
  ↓
เก็บเข้า "ภาษีซื้อเตรียมรายงาน"  → ใช้คำนวณ ภพ.30
```
**หมายเหตุ:** ยอดซื้อ / เจ้าหนี้ → เก็บทุกชื่อนิติใน sheet `SHOPS`
แต่ **ภาษีซื้อ (VAT)** → เก็บเฉพาะบิลชื่อ KLH เท่านั้น

## 2.2 ภาษีขาย (Output VAT)
```
แหล่ง: อีเมลธนาคารรายวัน (กรุงไทย ถุงเงิน + กรุงศรี)
  ↓
ยอดโอนเข้า = ฐานยอดขายเพื่อภาษี
  ↓
ยอดขายเพื่อภาษี = (ภาษีซื้อ) + (% ที่ผู้บริหารกำหนด)
                  ⚠️ ต้อง > ภาษีซื้อ เสมอ (มี VAT ต้องชำระ ไม่ขอคืน)
  ↓
ใช้ข้อมูลปีที่แล้ว เฉลี่ยรายเดือน เป็น baseline
```

## 2.3 หน้าผู้บริหารกรอกประมาณการ
```
- แสดง: ภาษีซื้อเดือนนี้ (จาก OCR)
- แสดง: ค่าเฉลี่ยยอดขายปีที่แล้ว (รายเดือน)
- ผู้บริหารกรอก: % เพิ่มเหนือภาษีซื้อ (เช่น +15%)
- ระบบคำนวณ: ยอดขายเป้าหมายเดือนนี้ + VAT ขายที่ต้องมี
- บันทึกลง sheet TAX_ESTIMATE
```

## 2.4 แจ้งเตือน LINE รายสัปดาห์
```
สิ้นสัปดาห์:
  เทียบ (ยอดโอนเข้าจริงสะสม) vs (เป้าหมายรายสัปดาห์ = เป้าเดือน ÷ 4)
  ↓
  IF ยอดจริง < เป้า → แจ้ง LINE: "ต้องฝากเงินเข้า KTB เพิ่ม X บาท เพื่อสร้างฐานภาษีขาย"
  IF ยอดจริง > เป้า → แจ้ง LINE: "ยอดเกินเป้า +X บาท"
```

## 2.5 ฟังก์ชันรายงานยอดขาย KLH (5 ข้อที่สั่ง)

### ข้อ 1: ยอดขายรวม POS → งบกำไรขาดทุน
```
รวมทุก transaction จาก POS (ทุกชื่อนิติ)
  → ยอดขายรวมทั้งร้าน
  → ใช้ในงบกำไรขาดทุน (P&L)
```

### ข้อ 2: แยกยอดขายสินค้า KLH ออกจาก POS
```
ในแต่ละบิล POS:
  filter รายการสินค้าที่ col AD (TAX_ENTITY) == "KLH"
  → รวมเฉพาะยอดสินค้า KLH ที่ขายไป
```

### ข้อ 3: ยอดขาย KLH จาก Statement (กรุงศรี + กรุงไทย)
```
ดึงยอด statement 2 ธนาคาร
  → ตั้งเป็นยอดขาย KLH
  → ออกรายงานเทียบ:  ยอด POS (ข้อ 2)  vs  ยอด Statement
  → "ใช้ทั้งคู่เทียบ" เพื่อหา gap
```

### ข้อ 4: รายงานขายสินค้า KLH (รับเข้าชื่อ KLH) + ตัดสต๊อก
```
สินค้าที่รับเข้าในชื่อ KLH (Invoice OCR + WMS)
  → สร้างรายงานขายจำลอง ให้ยอดขายล้อกับ statement
  → ตัดสต๊อกสินค้าตามยอดขายจำลองนั้น
```

### ข้อ 5: สิ้นเดือน — แยกสต๊อก KLH vs ร้านอื่น
```
สิ้นเดือนแสดง 2 ก้อนแยกกัน:
  A) สต๊อกสินค้า KLH คงเหลือ
  B) สต๊อกหน่วยเข้าร้านอื่น คงเหลือ
  ↓
สำหรับ KLH:
  สร้าง "ยอดขายสินค้า KLH จำลอง" (จาก statement)
  → นำไปหัก = สต๊อกคงเหลือ KLH ที่รายงานได้
```

## Step 1: รากฐานข้อมูล
- [ ] สร้าง sheet TAX_INPUT, TAX_OUTPUT, TAX_ESTIMATE, BANK_TRANSACTIONS
- [ ] ต่อ Invoice OCR → เขียน VAT ซื้อ KLH ลง TAX_INPUT (filter ชื่อ KLH)

## Step 2: อ่านอีเมลธนาคาร
- [ ] `fetchBankEmails()` กรุงไทย + กรุงศรี → BANK_TRANSACTIONS
- [ ] แยกโยกเงิน vs ยอดขาย → TAX_OUTPUT

## Step 3: หน้าประมาณการ สรรพากร
- [ ] `tax_estimate.html` — ภพ.30 (ซื้อ/ขาย/ชำระ)
- [ ] `tax_exec.html` — ผู้บริหารกรอก % + baseline ปีที่แล้ว

## Step 4: แจ้งเตือน LINE รายสัปดาห์
- [ ] เทียบยอดจริง vs เป้า → push LINE

## Step 5: รายงานยอดขาย KLH (5 ข้อ)
- [ ] ยอดขายรวม POS (ข้อ 1)
- [ ] แยกยอด KLH จาก AD (ข้อ 2)
- [ ] เทียบ POS vs Statement (ข้อ 3)
- [ ] ขายจำลอง + ตัดสต๊อก (ข้อ 4)
- [ ] สิ้นเดือนแยกสต๊อก KLH/ร้านอื่น (ข้อ 5)
